Semestrální práce ke kurzu 4IZ238 - Dodací list

Úvod

V této dokumentaci jsou popsány všechny XML soubory vytvořené pro zpracování semestrální práce ke kurzu 4IZ238. Předmětem bylo vytvoření XML dokumentu, schématu, schematronu, transformací do PDF a HTML dokumentu a nakonec samotná dokumentace v DOCBOOKU.

Shema XML

В моем проекте это документ по имени index.xsd предназначеный для шаблонизации и стандартизации дальнейших xml документов для работы в команде и выполняющий функцию лекала в качестве значений рассмотрим на премере участка кода "Artist" в качестве страны мы можем выбрать один из готовых вариантов как и "singingVoice" где зарание указаны виды голосов.

XML docoment

XML docoment это файл имеющий расширение .xml и построеный согласно подготовленной схеме, хранящий данные для наших дальнейших преобразований и построений наших документов. И построен по схеме где каждый пользователь может иметь на разных ресурсах имеющих подключение к нашей базе ряд листов в которых может быть сколько угодно групп, в которых может быть сколько угодно учасников и альбомов, а в альбомах может быть сколькоугодно песен, при этом каждый обьект имеет обязательное описание обьекта в структуре данных.

XSLT-HTML

Это документ имеющий инструкцию по преобразованию нашей схемы в .html страницы и стили для них. Для стилизизации наших страниц я использовал фреймворк Bootstrap по причине его удобства и скорости работы с ним, а также чистый CSS для более точной подстройки стилей и позиционирования. В качестве заполнения мною были применины группы катрочек для отображения списка доступных групп, а после отображения списка участников и альбомов. Данные выводяться в цикле для избежания повторения кода и упрощения его обслуживания. Верстка выполнена с запасом к расширению и применению в дальнейших проэктах. Реализации в предстоящих проектах на практике полученых знаний. Пример кода карточек приведён ниже.

XSLT-PDF

Это документ имеющий инструкцию по преобразованию нашей схемы в .pdf страницы и стили для них. Для стилизации были применены стандартные стили. После описания используемых шаблонов и кодировок следует описание конструкции документа и параметров его страниц. За описанием порядка страниц следует описание самих страниц и их контента.
